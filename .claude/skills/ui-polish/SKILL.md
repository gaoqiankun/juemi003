# Skill: UI 打磨轮

> 当收到"UI 打磨"、"视觉调整"、"布局优化"类任务时使用本 Skill。

## 打磨前必读

1. 确认本次改动范围：哪些页面？用户侧还是 Admin？还是全站？
2. 读 `.claude/rules/frontend.md` 确认当前设计规范
3. 读 `web/AGENTS.md` 确认页面结构

## 常见打磨类型

### 布局与间距

检查清单：
- [ ] 页面级间距是否统一 `gap-4`
- [ ] 卡片内分组是否 `gap-3`
- [ ] 卡片 padding 是否 `p-4`
- [ ] 操作按钮是否都是 `size="sm"`（h-8），无自定义 `h-*`
- [ ] Canvas 页是否正确使用负边距突破 shell

### 表格操作列

Admin 表格操作列规范（经过多轮修正，务必遵循）：
- 操作区用**单个 `<td>`**，内部 `flex items-center gap-2`
- 按钮始终渲染，通过 `disabled` 表达不可用状态（不要条件隐藏按钮）
- 错误信息用 `title` 属性 + `cursor-help`，不要在单元格内展开文本（避免撑高行高）
- 表头操作列：单列，不用 `colSpan`

### i18n 文案

- 改完中文文案必须同步改 `en.json`
- 改完英文文案必须同步改 `zh-CN.json`
- 用 `grep` 确认两个文件 key 一致

### 主题适配

- 暗色/亮色主题均需验证（`useTheme` hook）
- Viewer 背景、网格线、线框色需分别适配
- 玻璃面板用 `bg-surface-glass`（设计 token），不写死颜色值

## 验收清单

```bash
cd web && npm run build     # 零错误
```

- [ ] 视觉上无明显对齐/间距异常
- [ ] 暗色主题下没有白色/黑色硬编码背景
- [ ] 所有文案两个 i18n 文件均已更新
- [ ] 没有引入新的响应式前缀
- [ ] 修改的表格操作列遵循单列 flex 规范
